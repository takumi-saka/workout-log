# 筋トレ記録アプリ 要件定義

## 目的

Androidスマホでジム中に素早く筋トレ記録を入力できる、自分専用のローカル寄りアプリを作る。

最初はPWAとして作り、必要になったらCapacitorで自分用スマホアプリ化できる構造にする。

## 技術方針

- 初期実装はPWA
- 将来的にCapacitorでアプリ化できる構造を保つ
- スマホ利用を最優先
- Android想定
- 外部リリースはしない
- LLM API内蔵は初期スコープ外
- CSV出力とLLM貼り付け用サマリー出力を用意する

## 最重要方針

入力の速さを最優先する。

ジム中にアプリを開いて、今日のメニューを選び、重量とレップを最小タップで記録できることを重視する。

## 利用スタイル

- ジムでその場入力する
- スマホ専用でよい
- 基本オンラインでよい
- オフライン対応は必須ではないが、PWAとして後から対応できる余地を残す
- PC編集は初期スコープ外

## UI方針

- ダークモードのみ
- 日本語UI
- NotionとApple純正アプリの中間のような、静かで見やすいデザイン
- 派手なジム系UIにはしない
- 入力は直接入力とプラス/マイナスボタンの両方を用意する
- 重量の増減ボタンは2.5kg刻み
- レップの増減ボタンは1rep刻み

## コア機能

### 今日のメニュー選択

- Day1からDay4の4分割メニューを選んで記録を開始する
- 曜日固定ではあるが、実際には変動するため、ユーザーが今日のDayを選ぶ
- Day選択は大きな4枚カードではなく、横スライドできるコンパクトなトグル/チップUIにする

### 記録入力

- セッション日付は保存時に自動で入れる
- ユーザーが毎回日付を入力する必要はない
- 各種目は目標セット数ぶん最初から入力欄を表示する
- 追加セットもできる
- 重量の初期値は前回重量を入れる
- レップの初期値は目標レップ数を入れる
- 前回重量がない場合、重量は空欄にする
- 前回記録は種目名の下に小さく表示する
- 前回記録の詳細表示は必要になったら検討する
- マシンの空き具合などで当日だけ別種目にする可能性がある
- 当日やらない種目は「スキップ」として扱い、記録対象とDay総合指標から外せるようにする
- ウォームアップセットは初期スコープ外
- ドロップセット、失敗レップ、補助ありなどの特殊記録は初期スコープ外

### 記録項目

セットごと:

- 重量
- レップ数

種目ごと:

- 任意メモ

セッション全体:

- パンプ
- 出し切り感
- 疲労感
- 任意メモ

評価は10段階スライダーとメモで入力する。

体重入力は初期スコープ外。

### メニュー編集

アプリ内でメニューを後から変更できるようにする。

編集対象:

- Day名
- 種目名
- 部位カテゴリ
- 目標セット数
- 目標レップ数
- 種目メモ/方針
- 並び順
- 有効/無効

種目数は増減できるようにする。

### 履歴

- 日付別に過去セッションを見られる
- 種目別に過去記録を見られる
- 入力画面で前回記録を参照できる

### 進捗

- Day別に進捗を見る
- Dayを選ぶと、まずそのDayの最大重量合計の推移を表示する
- 最大重量合計は、その日に実施した各種目の最大重量を合計した値
- 当日だけ代替/スキップした種目は最大重量合計に含めない
- その下に、そのDayに含まれる各種目のグラフを一覧表示する
- 種目を1つ選ぶ方式ではなく、各種目グラフを縦に並べる
- 種目別グラフの代表値はその日の最大重量
- グラフ内に重量ラベルを小さく表示する
- グラフ下に日付軸を表示し、どの日の記録か分かるようにする
- グラフは将来的にピンチアウト/ズームまたは横スクロールで細かく見られるようにしたい

### 出力

- CSV出力
- LLM貼り付け用自然文サマリー出力
- 出力条件は期間、月、種目で選べるようにする
- JSON出力は初期必須ではないが、バックアップ用途として後から検討する

- 胸

- 脚
- 肩
- 二頭
- 三頭
- 前腕
- 体幹
- その他
